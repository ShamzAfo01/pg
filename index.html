Strengthening Human–AI Payment Experiences

Page 1 · Introduction

Context

PAYAI enables autonomous agents and humans to transact securely and continuously. The protocol is powerful, but the current website experience limits comprehension, conversion, and adoption. Most of the language and interaction is optimised for protocol insiders, not for the people who need to trust and use it.

Objective

This engagement focuses on a UX audit and redesign of the PAYAI web experience. The goal is to translate a complex AI-first payment protocol into a clear, human-first interface that also remains machine-friendly for autonomous agents.

UX Problems Identified

Where the Current Experience Breaks

Page 3 · Audit Summary

1. Human Comprehension Gap

The current messaging reads as if the primary audience is the protocol itself or seasoned agents—not humans. Concepts such as autonomous agents, payment codes, and protocol states are not translated into everyday language that gives users confidence.

2. Visual Language & Iconography

The illustrations and icons are visually appealing but largely decorative. They don’t explain what actions the user can take or how the payment journey unfolds from intent to confirmation.

3. Weak Information Hierarchy

Key questions—“What does this do?”, “Why should I trust it?”, “What happens next?”—are not answered in a clear hierarchy. Benefits, proof, and next steps are buried in abstract language.

4. Missing Onboarding & State Feedback

There is no guided path for a first-time visitor to understand how agents transact on their behalf. Errors are reported as codes (e.g., 42) instead of meaningful explanations such as “Payment authorisation incomplete”.

5. Mobile Experience Gaps

With more than four out of five visits coming from mobile, the layout, typography, and controls need to be engineered as mobile-first experiences, not shrunk desktop views.

Persona Strategy

Designing for Humans in an AI-First System

Page 4 · Personas

Primary Persona: The Human Operator

A tech-forward user who is comfortable with automation, crypto-adjacent tools, and AI products, but who still makes decisions based on clarity, trust, and predictable outcomes.

  • Wants to automate payments without losing control
  • Needs transparent explanations of what the agent will do
  • Responds to clear visuals, step-by-step flows, and friendly copy
  • Is unlikely to tolerate opaque errors or unexplained failures

Secondary (Internal) Persona: The Autonomous Agent

A conceptual persona representing the AI systems interacting with PAYAI. While not human, its needs shape the way the product is structured and documented.

  • Requires predictable state transitions and stable endpoints
  • Performs best when UI states map cleanly to backend states
  • Benefits from semantic HTML and structured data for crawling

The public UX will be centred on the human operator. The agent persona is used internally to validate that design changes do not break automated flows.

Recommendations

AI-Optimised UX: Key Recommendations

Page 5 · Design & Content

1. Human-First Messaging

  • Rewrite the homepage and core flows in benefit-led language.
  • Introduce a short “What is an autonomous payment agent?” explainer.
  • Add inline tooltips for any protocol-level terminology.

2. Visual & Interaction System

  • Replace abstract visuals with diagrams that show the payment journey: human → agent → network → confirmation.
  • Standardise iconography so each icon has a clear semantic meaning.
  • Introduce micro-interactions that confirm state changes in real time.

3. Mobile-First Layout

  • Re-engineer spacing, headings, and CTAs for small screens.
  • Use sticky bottom actions for key flows.
  • Optimise loading and content density to minimise bounce.

4. Agent & SEO Optimisation

  • Use semantic HTML and structured data to support AI and search agents.
  • Align UI states with API states for predictable automation.
  • Define an error dictionary that maps codes to human-readable explanations.
